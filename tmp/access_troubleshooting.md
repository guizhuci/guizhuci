# 服务访问故障排查指南

## ✅ 服务状态确认

服务器状态：**正常运行**
- 端口5000正在监听
- Next.js服务正在运行
- 本地curl测试正常（HTTP 200）

## 🔧 常见问题解决

### 问题1：连接被拒绝

**可能原因：**
- 你在远程服务器上，但浏览器在本地
- 浏览器配置了代理

**解决方案：**

**选项A：如果你在同一台机器上**
1. 确认没有使用VPN或代理
2. 尝试使用 `http://127.0.0.1:5000` 而不是 `http://localhost:5000`
3. 清除浏览器缓存（Ctrl+Shift+Delete）

**选项B：如果你通过SSH连接到远程服务器**
需要配置SSH端口转发：
```bash
ssh -L 5000:localhost:5000 your-server
```
然后在本地浏览器访问：`http://localhost:5000/admin`

**选项C：查看系统提供的预览链接**
某些平台（如Coze）会提供外部预览链接，请查看：
- 控制台输出
- 平台预览按钮
- 文档说明

### 问题2：页面空白或加载失败

1. 打开浏览器开发者工具（F12）
2. 查看Console标签的错误信息
3. 查看Network标签是否有红色失败的请求
4. 如果有错误，截图并反馈

### 问题3：CORS错误

如果是API调用失败，可能需要检查：
- API路径是否正确
- 是否有跨域配置

## 📱 移动端访问

如果在手机上访问：
1. 确保手机和电脑在同一WiFi网络
2. 使用电脑的局域网IP访问（如192.168.x.x:5000）

## 🎯 推荐做法

### 如果你在Coze/Coder平台：
1. 查找平台的"预览"或"预览URL"按钮
2. 点击预览按钮自动打开浏览器
3. 平台会自动处理网络配置

### 如果在本地开发环境：
1. 确认服务在本地运行
2. 直接访问 `http://127.0.0.1:5000/admin`
3. 如果仍无法访问，尝试重启浏览器

## 📞 需要帮助？

如果以上方法都无法解决，请提供：
1. 你的操作系统（Windows/Mac/Linux）
2. 你使用的浏览器（Chrome/Edge/Firefox）
3. 具体的错误信息或截图
4. 你是通过什么方式连接到服务器的（本地/SSH/远程桌面）

## ✅ 验证命令

在终端执行以下命令验证服务：
```bash
curl http://127.0.0.1:5000/admin
```

如果返回HTML内容，说明服务正常，问题出在浏览器端。
